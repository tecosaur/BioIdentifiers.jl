BioIdentifiers.jl

Structured and validated types for biological and biomedical identifiers.

Built on FastIdentifiers.jl, each type provides parsing (from shortcodes, prefixed forms, and URLs), canonical string output, persistent URL generation, and round-trip guarantees.

Identifier Types

Protein & Structure

BioIdentifiers.AFDB — Type
AFDB <: BioIdentifier

AlphaFold Database identifier for predicted 3-D protein models.

AlphaFold DB identifiers reference AI-predicted protein structures from DeepMind/EMBL-EBI. They follow the format AF-<UniProt>-F<number> where UniProt is a valid UniProt accession and the F-number represents the fragment/model number. Parsing may throw a MalformedIdentifier if the format is invalid.

Examples

julia> parse(AFDB, "AF-P69905-F1")
AFDB:AF-P69905-F1

julia> parse(AFDB, "https://alphafold.com/entry/AF-Q9Y6K5-F1")
AFDB:AF-Q9Y6K5-F1
source
BioIdentifiers.PDB — Type
PDB <: BioIdentifier

A Protein Data Bank (PDB) structure identifier.

Managed by the Worldwide Protein Data Bank (wwPDB), the PDB is the single global archive of experimentally determined 3D structures of biological macromolecules. Identifiers are 4-character alphanumeric codes (digits and uppercase letters), with the first character always a digit.

Examples

julia> parse(PDB, "1A3N")
PDB:1A3N

julia> parse(PDB, "7QDH")
PDB:7QDH
source
BioIdentifiers.UniProt — Type
UniProt <: BioIdentifier

A UniProt (Universal Protein Resource) accession for protein sequences.

Maintained by the UniProt Consortium (EMBL-EBI, SIB, PIR), UniProt is the most comprehensive resource for protein sequence and functional annotation. Accessions come in two formats: 6-character Swiss-Prot entries (e.g. P04637) starting with O/P/Q, and 10-character TrEMBL entries (e.g. A0A0G2JMC8) starting with A–N or R–Z.

Examples

julia> parse(UniProt, "P04637")
UniProt:P04637

julia> parse(UniProt, "A0A0G2JMC8")
UniProt:A0A0G2JMC8

julia> parse(UniProt, "https://www.uniprot.org/uniprot/Q9Y6K5")
UniProt:Q9Y6K5
source
BioIdentifiers.UniRef — Type
UniRef <: BioIdentifier

UniProt Reference Clusters identifier.

UniRef provides clustered sets of sequences from the UniProt Knowledgebase at 50%, 90%, and 100% sequence identity. These clusters are widely used by AlphaFold, MMseqs2, and other bioinformatics tools to reduce redundancy. Identifiers follow format UniRef<level>_<UniProt> where level is 50, 90, or 100. Parsing may throw a MalformedIdentifier if the format is invalid.

Examples

julia> parse(UniRef, "UniRef50_Q9Y6K5")
UniRef:UniRef50_Q9Y6K5

julia> parse(UniRef, "UniRef90_P69905")
UniRef:UniRef90_P69905
source
BioIdentifiers.IntAct — Type
IntAct <: BioIdentifier

IntAct protein interaction record identifier (EBI).

IntAct provides a freely available, open source database system and analysis tools for molecular interaction data. All interactions are derived from literature curation or direct user submissions. Identifiers follow format EBI-<number>. Parsing may throw a MalformedIdentifier if the format is invalid.

Examples

julia> parse(IntAct, "EBI-123456")
IntAct:EBI-123456

julia> parse(IntAct, "https://www.ebi.ac.uk/intact/interaction/EBI-789012")

source
BioIdentifiers.InterPro — Type
InterPro <: BioIdentifier

An InterPro protein family and domain identifier.

Maintained by EMBL-EBI, InterPro integrates protein signatures from Pfam, PRINTS, PROSITE, SMART, and other member databases into a single classification resource. Identifiers have the format IPR followed by 6 zero-padded digits.

Examples

julia> parse(InterPro, "IPR011009")
InterPro:IPR011009

julia> purl(parse(InterPro, "IPR011009"))
"https://www.ebi.ac.uk/interpro/entry/InterPro/IPR011009"
source
BioIdentifiers.Pfam — Type
Pfam <: BioIdentifier

A Pfam protein family identifier.

Originally developed at the Sanger Institute and now hosted by EMBL-EBI as part of InterPro, Pfam classifies protein sequences into families and domains using hidden Markov model profiles. Identifiers have the format PF followed by 5 zero-padded digits.

Examples

julia> parse(Pfam, "PF00069")
Pfam:PF00069

julia> purl(parse(Pfam, "PF00069"))
"https://www.ebi.ac.uk/interpro/entry/pfam/PF00069"
source
BioIdentifiers.PXD — Type
PXD <: BioIdentifier

A ProteomeXchange dataset identifier for mass spectrometry proteomics data.

ProteomeXchange is a consortium providing a single point of submission for proteomics datasets to partner repositories (PRIDE, PeptideAtlas, MassIVE, jPOST). Both PXD (original) and RPXD (reprocessed) prefixes are accepted during parsing; the canonical output uses PXD. Identifiers consist of the prefix followed by 6 zero-padded digits.

Examples

julia> parse(PXD, "PXD000001")
PXD:PXD000001

julia> parse(PXD, "RPXD123456")
PXD:PXD123456

NCBIGene <: BioIdentifier

NCBI Entrez Gene numeric primary key.

Entrez Gene provides a unified query environment for genes defined by sequence and/or in Map Viewer. NCBIGene:5594

BioIdentifiers.RRID — Type
RRID <: BioIdentifier

A Research Resource Identifier (RRID) for research materials and tools.

Managed by SciCrunch and promoted by the RRID Initiative, RRIDs provide persistent unique identifiers for antibodies, cell lines, organisms, plasmids, software tools, and other research resources. Identifiers follow the format RRID:<provider>_<id>, where the provider code indicates the source registry (e.g. AB for antibodies, SCR for software).

PubChem <: BioIdentifier

A PubChem identifier for chemical compounds, substances, and bioassays.

Operated by NCBI, PubChem is the world's largest freely accessible chemical database, providing information on chemical structures, properties, biological activities, and safety data. Identifiers use a prefix indicating the record type — CID (compound), SID (substance), or AID (bioassay) followed by a positive integer.

Examples

julia> parse(PubChem, "CID2244")
PubChem:CID2244

julia> parse(PubChem, "SID12345678")
PubChem:SID12345678

BioIdentifiers.Reactome — Type
Reactome <: BioIdentifier

A Reactome curated biological pathway identifier.

Reactome is a free, open-source, peer-reviewed pathway knowledgebase maintained by OICR, NYU Langone, and EMBL-EBI. Identifiers follow the format R-<species>-<number>, where the 3-letter species code indicates the organism (e.g. HSA for human, MMU for mouse).

Examples

julia> parse(Reactome, "R-HSA-199420")
Reactome:R-HSA-199420

julia> parse(Reactome, "https://reactome.org/PathwayBrowser/#/R-MMU-123456")
Reactome:R-MMU-123456

BioIdentifiers.NCBITaxon — Type
NCBITaxon <: BioIdentifier

NCBI Taxonomy species/strain identifier.

The NCBI Taxonomy database contains the names of all organisms that are represented in the genetic databases with at least one nucleotide or protein sequence. The taxonomy database includes phylogenetic and taxonomic information for the organisms. Identifiers are positive integers. Parsing may throw a MalformedIdentifier if the format is invalid.

NCBITaxon:9606
